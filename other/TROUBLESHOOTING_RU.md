# Устранение проблем

## Ошибка "не доступно" или "cannot bind"

### Причины:

1. **Порт уже используется**
   - Другое приложение уже использует порт 1443
   - Предыдущий экземпляр прокси еще работает

   **Решение:**
   ```bash
   # Проверить, какое приложение использует порт
   netstat -ano | findstr :1443
   
   # Использовать другой порт
   tg-ws-proxy.exe --port 8443
   ```

2. **Недостаточно прав**
   - Порты ниже 1024 требуют прав администратора
   - Брандмауэр блокирует доступ

   **Решение:**
   - Запустите от имени администратора
   - Добавьте исключение в брандмауэр Windows

3. **Неверный адрес**
   - IP-адрес не существует на этом компьютере

   **Решение:**
   ```bash
   # Слушать на всех интерфейсах
   tg-ws-proxy.exe --host 0.0.0.0 --port 1443
   
   # Или только локально
   tg-ws-proxy.exe --host 127.0.0.1 --port 1443
   ```

## GUI не запускается

### Причины:

1. **Отсутствуют графические драйверы**
   - Устаревшие драйверы видеокарты
   - Работа через удаленный рабочий стол без поддержки OpenGL

   **Решение:**
   - Обновите драйверы видеокарты
   - Используйте версию без GUI:
   ```bash
   cargo build --release --no-default-features
   ```

2. **Ошибка инициализации окна**
   - Проблемы с системой окон Windows

   **Решение:**
   - Проверьте логи в папке `logs/`
   - Прокси продолжит работать в фоне

## Логи не создаются

### Причины:

1. **Нет прав на запись**
   - Папка защищена от записи

   **Решение:**
   - Запустите от имени администратора
   - Переместите программу в папку с правами записи (например, Documents)

2. **Диск заполнен**
   - Недостаточно места на диске

   **Решение:**
   - Освободите место на диске
   - Удалите старые логи из папки `logs/`

## Telegram не подключается

### Причины:

1. **Неверная ссылка прокси**
   - Используется локальный IP (127.0.0.1) на другом устройстве

   **Решение:**
   ```bash
   # Узнайте ваш локальный IP
   ipconfig
   
   # Запустите с правильным IP
   tg-ws-proxy.exe --host 0.0.0.0 --link-ip 192.168.1.100
   ```

2. **Брандмауэр блокирует соединения**
   - Windows Firewall блокирует входящие подключения

   **Решение:**
   - Добавьте правило в брандмауэр:
   ```
   Панель управления → Брандмауэр Windows → Дополнительные параметры
   → Правила для входящих подключений → Создать правило
   → Порт → TCP → 1443 → Разрешить подключение
   ```

3. **Неверный секрет**
   - Секрет в ссылке не совпадает с секретом прокси

   **Решение:**
   - Скопируйте ссылку из GUI (кнопка "Copy")
   - Или проверьте секрет в логах

## Высокое использование ресурсов

### Причины:

1. **Слишком много соединений**
   - Превышен лимит файловых дескрипторов

   **Решение:**
   ```bash
   # Ограничьте количество соединений
   tg-ws-proxy.exe --max-connections 100
   ```

2. **Утечка памяти в логах**
   - Слишком много сообщений в памяти GUI

   **Решение:**
   - Перезапустите приложение
   - Используйте версию без GUI для длительной работы

## Проверка работоспособности

### Шаг 1: Проверьте логи
```bash
# Откройте последний лог
notepad logs\tg-ws-proxy.log.2026-04-08
```

### Шаг 2: Проверьте порт
```bash
# Проверьте, что порт слушается
netstat -ano | findstr :1443
```

### Шаг 3: Проверьте подключение
```bash
# Попробуйте подключиться локально
telnet localhost 1443
```

## Получение помощи

Если проблема не решена:

1. Соберите информацию:
   - Версия Windows
   - Содержимое логов из `logs/`
   - Точный текст ошибки
   - Команда запуска

2. Проверьте логи на наличие ERROR и WARN сообщений

3. Попробуйте запустить с отладкой:
   ```bash
   tg-ws-proxy.exe --verbose
   ```

## Быстрые решения


```bash
cargo build --release --no-default-features
```
